# LTB Engine – Current Development Status

## Overview

This document summarizes the current development state of the LTB (Live Trading Bot) trading engine.

The system is implemented as a multi-worker event-driven architecture where independent workers communicate through an internal event bus.

Major subsystems include:

- Market data ingestion
- Indicator computation
- Strategy evaluation
- Signal filtering and ranking
- Portfolio optimization
- Execution engine
- Risk management
- Runtime monitoring

---

# Architecture Overview

The trading engine operates through the following layered pipeline:

Market → Indicator → Strategy → Signal Pipeline → Portfolio Optimization → Execution → Risk

All components communicate using an internal event bus.

Each worker subscribes only to relevant events.

# Signal Processing Pipeline

Signals generated by strategies pass through several filtering stages.

Workers

- SignalDedupWorker
- SignalPersistenceWorker
- SignalRankingWorker
- LiquidityFilterWorker
- StrategyAllocationWorker

Signal flow:

strategy.signal  
→ dedup.signal  
→ ranked.signal  
→ allocation.signal  
→ optimized.signal

Purpose

- remove duplicate signals
- verify signal persistence
- rank signals using alpha scoring
- filter low liquidity assets
- allocate capital across strategies

# Current System Characteristics

The engine has the following design characteristics.

Event-driven architecture

All subsystems communicate through a message bus.

Multi-worker concurrency

Each subsystem operates independently.

Strategy modularity

New strategies can be added without modifying the core engine.

Layered pipeline

Signal generation, filtering, allocation, and execution are separated.
